# Frontend Structure & Naming

The folder taxonomy is **closed**: do not add new top-level folders under
`src/frontend/src/` without explicit user approval. Place new code in the
folder that already owns the concern.

## Top level (`src/frontend/src/`)

```
src/frontend/src/
├── app.html
├── app.d.ts
├── custom-events.d.ts
├── global.d.ts
├── hooks.ts
│
├── routes/                 SvelteKit file-based routing
│   ├── (app)/              Authenticated wallet shell (homepage, activity,
│   │                       earn, earning, explore, nfts, rewards, settings,
│   │                       transactions, wc)
│   ├── (public)/           Pages without auth
│   ├── (sign)/             Signer flow (separate target, see SIGNER_DOMAINS.md)
│   ├── +layout.svelte
│   ├── +layout.ts
│   └── sitemap.xml
│
├── lib/                    Cross-cutting domain (shared across chains)
│   ├── components/         UI components grouped by feature (auth, send,
│   │                       receive, swap, contact, hero, ai-assistant, …
│   │                       and `core/`, `common/`, `ui/`, `icons/`, `loaders/`)
│   ├── services/           Side-effectful operations (services calling api,
│   │                       canisters, providers, rest, scheduling)
│   ├── stores/             Svelte stores (writable / readable / certified)
│   ├── derived/            Derived stores (Svelte) and rune-derived modules
│   ├── api/                Hand-written wrappers around generated declarations
│   ├── canisters/          IC actor factories
│   ├── providers/          Long-lived providers (auth client, swap providers, …)
│   ├── rest/               REST client wrappers (CoinGecko, Blockstream, …)

│   ├── schema/             Zod schemas for typed boundaries
│   ├── validation/         Validation helpers (assertions, parsers)
```

## Naming conventions

These are **strict**. ESLint / svelte-check do not enforce all of them,
so agents must.

### File suffixes

| Suffix            | Meaning                                                                               | Example                                   |
| ----------------- | ------------------------------------------------------------------------------------- | ----------------------------------------- |
| `*.svelte`        | Component (PascalCase filename)                                                       | `AiAssistantChat.svelte`                  |
| `*.svelte.ts`     | Module that uses Svelte 5 runes (`$state`, `$derived`, `$effect`) outside a component | `theme.svelte.ts`                         |
| `*.services.ts`   | Side-effectful module (calls APIs, toasts, mutates state)                             | `exchange.services.ts`                    |
| `*.utils.ts`      | Pure helpers — no I/O, no side effects, no DOM access                                 | `format.utils.ts`, `network.utils.ts`     |
| `*.constants.ts`  | Compile-time constants, enums, lookup tables                                          | `tokens.constants.ts`, `app.constants.ts` |
| `*.derived.ts`    | Derived Svelte stores or rune-derived modules                                         | `tokens.derived.ts`, `auth.derived.ts`    |
| `*.store.ts`      | Writable / readable Svelte store factory                                              | `i18n.store.ts`, `auth.store.ts`          |
| `*.api.ts`        | Hand-written wrapper over a generated declaration                                     | `backend.api.ts`                          |
| `*.canister.ts`   | Class wrapping IC actor calls                                                         | `signer.canister.ts`                      |
| `*.provider.ts`   | Long-lived provider (auth client, swap provider, …)                                   | `auth-client.providers.ts`                |
| `*.rest.ts`       | REST client                                                                           | `coingecko.rest.ts`                       |
| `*.scheduler.ts`  | Worker scheduling primitive                                                           | `nft.scheduler.ts`                        |
| `*.worker.ts`     | Web worker entrypoint                                                                 | `exchange.worker.ts`                      |
| `*.env.ts`        | Environment / feature flag / catalog under `env/`                                     | `tokens.usdc.env.ts`                      |
| `*.spec.ts`       | Vitest spec — under `src/frontend/src/tests/` mirroring source                        | `exchange.services.spec.ts`               |
| `*.test-utils.ts` | Reusable test helper under `tests/utils/`                                             | `derived.test-utils.ts`                   |
| `*.mock.ts`       | Mock data / factories under `tests/mocks/`                                            | `exchanges.mock.ts`                       |

### Casing

| Thing                         | Style             | Example                            |
| ----------------------------- | ----------------- | ---------------------------------- |
| `.svelte` filename            | `PascalCase`      | `AiAssistantChat.svelte`           |
| `.ts` / `.svelte.ts` filename | `kebab-case`      | `exchange.services.ts`             |
| Folder                        | `kebab-case`      | `ai-assistant/`, `wallet-connect/` |
| Component name                | `PascalCase`      | `AiAssistantChat`                  |
| TS type / interface           | `PascalCase`      | `BackendExchangeRate`              |
| Function / variable           | `camelCase`       | `exchangeRateUsdToCurrency`        |
| Constant export               | `SCREAMING_SNAKE` | `BSC_MAINNET_NETWORK`              |
| Test ID / data attribute      | `kebab-case`      | `data-tid="ai-assistant-chat"`     |

### Imports — aliases only

The local rule `local-rules/no-relative-imports` is **`error`** under
`src/frontend/src/**`. Use the path aliases declared in
[`svelte.config.js`](../../../svelte.config.js):

| Alias           | Path                                                                 |
| --------------- | -------------------------------------------------------------------- |
| `$lib`          | `src/frontend/src/lib`                                               |
| `$routes`       | `src/frontend/src/routes`                                            |
| `$btc`          | `src/frontend/src/btc`                                               |
| `$eth`          | `src/frontend/src/eth`                                               |
| `$evm`          | `src/frontend/src/evm`                                               |
| `$icp`          | `src/frontend/src/icp`                                               |
| `$sol`          | `src/frontend/src/sol`                                               |
| `$icp-eth`      | `src/frontend/src/icp-eth`                                           |
| `$env`          | `src/frontend/src/env`                                               |
| `$declarations` | `src/declarations`                                                   |
| `$tests`        | `src/frontend/src/tests` (test-only, declared in `vitest.config.ts`) |

```ts
import { i18n } from '$lib/stores/i18n.store';
import type { TokenId } from '$declarations/backend/backend.did';
import { mockIdentity } from '$tests/mocks/identity.mock';
import { exchangeRates } from '$lib/derived/exchange.derived';
```

### Forbidden imports / patterns

- Relative imports across folders (`../../...`) under
  `src/frontend/src/**` (eslint).
- Hand-edits to `src/declarations/**` (regenerate via `npm run generate`).
- Hand-edits to `src/frontend/src/lib/types/i18n.d.ts` (run `npm run i18n`).
- Hand-edits to `src/frontend/src/env/tokens/tokens.{sns,ckerc20,icrc,ext}.json`
  (regenerated by the `build:tokens-*` scripts and the `update-tokens`
  workflow).
- Unprompted edits to non-`en` locale JSON files. The `auto-update-i18n`
  workflow only syncs key structure (existing values are preserved), so
  hand-edits aren't auto-overwritten — but translations should only be
  authored when the developer explicitly asks. See
  [`i18n-and-a11y.md`](./i18n-and-a11y.md#i18n).
- `0n` literal — use the shared `ZERO` constant (eslint).
- `console.error` / `console.warn` — use `consoleError` / `consoleWarn`
  from `$lib/utils/console.utils` (eslint).

## Where to put new files (decision tree)

1. **Is it a route / page?** → `src/frontend/src/routes/(app|public|sign)/...`
2. **Is it a generic UI primitive (Button, Modal, Tooltip)?** → look in
   `$lib/components/ui/` first; only add a new primitive if it doesn't
   already cover it.
3. **Is it a small, app-wide reusable presentational block (List,
   ListItem, Divider)?** → `$lib/components/common/`.
4. **Is it specific to a feature (send, swap, ai-assistant, contact,
   hero, …)?** → `$lib/components/<feature>/...`.
5. **Is it specific to one chain (BTC, ETH, EVM, ICP, SOL, ICP-ETH)?** →
   `$btc/components/<feature>/...` (and so on for the other chains).
6. **Is it a side-effectful operation?** → `*.services.ts` under the
   matching `services/` folder.
7. **Is it a pure helper?** → `*.utils.ts` under the matching `utils/`
   folder. Cross-chain → `$lib/utils/`. Chain-specific → `<chain>/utils/`.
8. **Is it a constant / enum?** → `*.constants.ts` in the matching
   `constants/`.
9. **Is it about which networks / tokens are supported / default?** →
   `$env/`. See
   [`workflows/new-token-or-network.md`](./workflows/new-token-or-network.md).
10. **Is it a Svelte store factory or derived store?** → `<area>/stores/`
    or `<area>/derived/` respectively.
11. **Is it a Svelte 5 rune-state module?** → suffix `.svelte.ts` and put
    it next to its consumers (or in a `stores/` folder if shared).
12. **Is it generated?** → don't create it by hand. Run the generator.
13. **None of the above?** → ask. Don't invent a folder.
